# 微信小程序自动化错误捕获实施总结

## 一、核心问题分析

### 1.1 主要问题：automator.launch() 无法启动

**问题表现**：
```
Error: Failed to launch wechat web devTools, please make sure http port is open
```

**根本原因**：
1. **微信开发者工具已经在运行** - `automator.launch()` 会尝试重新启动CLI，与已运行的实例冲突
2. **端口配置不匹配** - 手动开启的服务端口（59092）与automator期望的端口不一致
3. **启动方式错误** - 直接使用 `automator.launch()` 需要微信开发者工具完全关闭

### 1.2 解决方案

**最终可行方案**：使用CLI手动启动自动化模式 + automator.connect()

```javascript
// 1. 通过CLI启动自动化模式
const cliProcess = spawn(cliPath, [
  'auto',
  '--project', projectPath,
  '--auto-port', '9420'  // 使用默认端口9420
]);

// 2. 等待10秒让自动化模式完全启动
await new Promise(resolve => setTimeout(resolve, 10000));

// 3. 使用connect连接（而不是launch）
const miniProgram = await automator.connect({
  wsEndpoint: 'ws://localhost:9420'
});
```

## 二、关键技术点

### 2.1 端口配置

| 端口类型 | 端口号 | 用途 |
|---------|--------|------|
| 服务端口 | 59092（或其他） | 微信开发者工具的HTTP服务端口 |
| 自动化端口 | 9420（默认） | automator的WebSocket连接端口 |

**注意**：这两个端口是不同的！
- 服务端口：在"设置 → 安全设置"中配置
- 自动化端口：通过CLI的`--auto-port`参数指定

### 2.2 API方法差异

测试中发现部分API方法不可用，需要使用正确的方法：

| 功能 | ❌ 不可用 | ✅ 可用 |
|------|----------|---------|
| 获取页面路径 | `await page.path()` | `page.path` (属性) |
| 获取DOM样本 | `page.$$eval()` | 遍历元素调用`element.attribute()` |
| 页面截图 | `page.screenshot()` | `miniProgram.screenshot()` |

### 2.3 成功捕获的数据

**1. 页面信息**
```javascript
const page = await miniProgram.currentPage();
const pagePath = page.path;  // "pages/home/home"
const pageData = await page.data();  // 页面数据对象
```

**2. DOM结构**
```javascript
const views = await page.$$('view');  // 90个元素
for (const view of views) {
  const className = await view.attribute('class');
  const id = await view.attribute('id');
}
```

**3. 页面截图**
```javascript
const screenshot = await miniProgram.screenshot();  // Buffer
const base64 = screenshot.toString('base64');  // 259896字符
// 可直接发送给AI进行图像分析
```

**4. Console日志**
```javascript
miniProgram.on('console', msg => {
  if (msg.type === 'error') {
    // 捕获错误日志
    console.log(msg.text);
  }
});
```

## 三、实施注意事项

### 3.1 环境配置要求

**必须完成的配置**：

1. **微信开发者工具设置**
   - 设置 → 安全设置 → 开启"服务端口"
   - 设置 → 通用设置 → 开启"允许命令行调用"

2. **CLI路径确认**
   ```bash
   /Applications/wechatwebdevtools.app/Contents/MacOS/cli
   ```

3. **项目路径**
   - 必须是绝对路径
   - 确保项目可以正常打开

### 3.2 启动流程

**正确的启动顺序**：

```
1. 关闭微信开发者工具（如果已打开）
   ↓
2. 通过CLI启动自动化模式
   cli auto --project <path> --auto-port 9420
   ↓
3. 等待10秒（确保完全启动）
   ↓
4. 使用automator.connect()连接
   ↓
5. 开始捕获数据
```

### 3.3 常见错误处理

**错误1：连接失败**
```
Failed connecting to ws://localhost:9420
```
**解决**：
- 确认CLI auto命令已成功启动（查看输出是否有"✔ auto"）
- 延长等待时间（从5秒增加到10秒）
- 检查端口是否被占用：`lsof -i :9420`

**错误2：方法不存在**
```
page.screenshot is not a function
```
**解决**：
- 使用 `miniProgram.screenshot()` 而不是 `page.screenshot()`
- 参考上面的API方法差异表

**错误3：微信工具未响应**
```
Failed to launch wechat web devTools
```
**解决**：
- 完全退出微信开发者工具
- 使用CLI启动而不是手动打开
- 不要使用 `automator.launch()`，改用 `automator.connect()`

### 3.4 性能考虑

1. **启动时间**：首次启动需要10-15秒
2. **截图大小**：约250KB，Base64约260KB
3. **DOM元素**：根据页面复杂度，可能有几十到几百个元素
4. **内存占用**：automator会保持WebSocket连接，注意及时断开
